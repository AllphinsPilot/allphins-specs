# CLAUDE.md — allphins-specs

Executable behaviour specs at `specs/<area>/<ID>.md`. Human reference: `README.md`.

## Guardrails (always apply)

- NEVER hand-create, rename, or delete a spec file, and never invent an ID. Use the scripts.
- NEVER hand-edit any `index.md` — they are generated by `.github/scripts/generate_indexes.py`.
- The path is the identity: `<area>` = lowercased ID prefix, `<ID>` = `PREFIX-NNN`.
  Front-matter must satisfy `spec.schema.json`.
- ONE scenario per spec. A spec has exactly one `## Scenario:`; parameterise with an
  `### Examples` table, never a second scenario. A multi-scenario behaviour (a flow)
  becomes its own area, one spec per scenario (e.g. `book-003/`). Enforced by
  `.github/scripts/validate_specs.py`.

## Spec operations → use the `managing-specs` skill

For creating, deleting, or running specs, follow the **managing-specs** skill
(`.claude/skills/managing-specs/SKILL.md`). In short:

- Create: `python .github/scripts/new_spec.py --area <area> --title "<title>"`
- Delete: `python .github/scripts/delete_spec.py <ID>`
- After a title/spec-set change: `python .github/scripts/generate_indexes.py` (verify with `--check`)
- Validate structure: `python .github/scripts/validate_specs.py`
