## Objective

Prove a Lakehouse scenario list refuses creation unless Peril is included as a filter, and prevents selecting Peril a second time.

## Preconditions

- The user is creating a scenario list on the Aggregations page.

## Scenario: Lakehouse scenario lists require Peril as a filter

```gherkin
Given a Lakehouse line of business
When the user tries to create a scenario list whose only filter is not Peril
Then creation is refused because Peril is an obligatory filter
And once Peril is chosen, Peril cannot be selected again for a second filter
```
