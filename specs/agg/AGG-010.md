## Objective
Prove the "See scenario definition" panel shows coherent, non-editable metadata and that "View aggregation" navigates to the computation page.

## Preconditions
- A scenario list with at least one scenario exists.

## Scenario: definition metadata is shown read-only
```gherkin
Given a scenario list is open
When the user clicks "See scenario definition"
Then the left panel shows coherent scenario metadata (name, risk-group filters)
And none of the displayed metadata can be edited
```

## Scenario: View aggregation opens the computation page
```gherkin
Given the scenario definition panel is open
When the user clicks "View aggregation"
Then the computation page opens with coherent information for that scenario
```
