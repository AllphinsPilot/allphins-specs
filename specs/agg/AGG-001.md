## Objective
Prove each aggregation object type can be created, and that when at least one data source has been delivered the created scenarios carry data.

## Preconditions
- A portfolio (or group) is available on the Aggregations page.
- At least one data source has been delivered (for the data assertion).

## Scenario: each scenario type can be created
```gherkin
Given the Aggregations page is open
When the user creates a Scenario list, a Scenario, and a TZ report
Then each is created and listed
And when a data source has been delivered, the created scenarios show exposure data
```

## Assumptions
- Source says "in case at least one data source is delivered, ensure that the scenarios have data"; with no data source the data assertion is skipped. Confirm the empty-data-source behaviour is "created but empty", not an error.
