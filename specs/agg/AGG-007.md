## Objective
Prove the view (eye) entry point opens a scenario read-only and prevents saving changes.

## Preconditions
- A scenario exists in a scenario list.

## Scenario: viewing a scenario is read-only
```gherkin
Given a scenario list with a scenario
When the user clicks the eye button next to the scenario name
Then the scenario opens in view mode
And any attempted change cannot be saved
```
