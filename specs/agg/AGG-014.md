## Objective
Prove that a loss-based custom scenario on Casualty cannot be created unless both Insured and Subclass are specified.

## Preconditions
- A non-Lakehouse line of business.
- The user is creating a loss-based Custom scenario on the Aggregations page for Casualty.

## Scenario: Casualty loss-based custom scenario requires Insured and Subclass
```gherkin
Given a loss-based Custom scenario is being created on Casualty
When Insured and Subclass are not both specified
Then the scenario cannot be created
And once Insured and Subclass are both specified, the scenario can be created
```

## Assumptions
- Source row 122 marks this rule as "true after the 27/03 POD release"; this spec assumes the rule is in effect for the version under test. Confirm the release gate.
